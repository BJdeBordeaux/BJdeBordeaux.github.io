# 博客发布流程注意事项

> 每次发文章前看一眼，避免踩坑。

---

## 发布流程

1. 在 `articles/` 下创建一个 `.md` 文件，**必须带 front matter**
2. 运行 `python3 generate_article.py "文件名关键词"` 生成 HTML
3. **手动** 在 `index.html` 的 `article-list` 里添加新文章卡片（插入到最前面）
4. 如果有新分类，在 `index.html` 的 `filter-bar` 里加对应按钮
5. `git add -A && git commit && git push origin master`

---

## 已踩过的坑

### 1. MD 文件必须带 front matter，否则 HTML 标题显示"无标题"

front matter 格式：
```yaml
---
title: 文章标题
subtitle: 副标题（可选）
tag: 分类名
date: 2026年5月
desc: OG摘要，截取到第一个中文句末标点
image: 封面图路径（可选）
---
```

### 2. 不要创建"XXX_改写版.md"之类的副本

`generate_article.py` 按文件名关键词匹配，如果存在多个匹配项，可能匹配到错误的文件。**改写直接覆盖原文件**，不要留副本。

### 3. HTML 文件名由 title 决定，不是由 MD 文件名决定

`generate_article.py` 用 `slugify(title)` 生成 HTML 文件名。改了 title 之后，HTML 文件名也会变，首页卡片的 `href` 要同步更新。

### 4. 首页文章卡片是手写死的

`index.html` 里的文章列表不是自动生成的。每次发新文章，必须手动在 `<div class="article-list">` 最前面插入新卡片。格式参考现有卡片。

### 5. 时区

- 在法国 → 巴黎时间（CET/CEST）
- 在国内 → 东八区
- 碎碎念时间戳备注时区，方便改

### 6. 图片处理

- 配图用 `convert_to_webp.py` 转 WebP，存到 `images/moments/`
- 碎碎念图片在 `style.css` 里已改为按原始比例显示
